# 15秒记忆现场

《15秒记忆现场》是一个手机竖屏网页小游戏原型。玩家需要观察一段 15 秒的便利店监控画面，记住画面里发生的变化，然后回答一道关于刚才细节的问题。

当前版本是可试玩原型，不是商业完整版本。它已经具备完整的首页、观察、答题、反馈、结算流程，并加入了难度递增、监控录像风格 UI、事件日志、问题生成和开发调试面板。

## 核心玩法

1. 点击“开始观察”。
2. 观察 15 秒便利店监控画面。
3. 画面中会发生顾客、商品、灯光、时钟、收银台、冰柜、监控异常等变化。
4. 观察结束后回答一道问题。
5. 答对加 10 分，答错扣 1 条生命。
6. 生命归零后进入结算页，根据成功完成轮数给出评价。

## 当前功能

- 手机竖屏优先的网页小游戏界面。
- 便利店监控画面，使用 DOM 和 CSS 分层搭建。
- 15 秒观察倒计时。
- 多种事件：
  - 顾客进店
  - 顾客离店
  - 商品被拿走
  - 货架商品消失
  - 冰柜数量变化
  - 时钟变化
  - 收银金额变化
  - 灯光闪烁
  - 监控屏异常影子
- 难度递增：
  - 第 1-2 轮：3 个简单事件
  - 第 3-5 轮：5 个事件，加入中等事件
  - 第 6 轮以后：7 个事件，加入异常事件和顺序类问题
- 问题基于本局真实 `eventLog` 生成。
- 四选一答题。
- 答题反馈、生命、分数、结算评级。
- 右下角 Debug 面板，可查看 `sceneState`、`eventLog` 和当前问题。
- 基础稳定性处理：防止快速点击导致重复结算或跳轮。

## 当前视觉风格

项目采用轻微诡异的监控录像风格：

- 暗色背景
- REC / CAM 标识
- 监控时间戳
- 扫描线和轻微雪花噪点
- 暗角
- 观察结束时淡出变暗
- 物品变化时对应物品高光闪烁
- 顾客离店时有向门移动并淡出的动画

画面仍然保持极简原型风，不使用真实美术素材。

## 如何运行

### 方式一：直接打开

直接在浏览器中打开：

```text

```

在项目目录运行：


然后访问：

```text
http://localhost:8000
```

也可以使用 VS Code Live Server 等静态服务器工具。

## 核心模块概览

这些模块当前都在 `script.js` 中：

- `DataConfig`：集中保存顾客、商品、事件、难度、文案等配置。
- `DifficultyManager`：根据轮数返回事件数量、允许事件类型、允许问题类型和观察时长。
- `RatingManager`：根据成功完成轮数返回结算评级。
- `EventGenerator`：根据难度生成本轮事件列表。
- `TimelinePlayer`：按时间播放事件，并同步更新画面和日志。
- `QuestionGenerator`：根据本局 `eventLog` 生成问题。
- `SceneRenderer`：根据 `sceneState` 渲染便利店分层 DOM 场景。
- `GameState`：保存分数、生命、轮数、阶段、倒计时、当前问题等游戏状态。
- `UIController`：负责页面切换、状态栏、问题、反馈和结算页渲染。
- `DebugPanel`：开发调试面板，默认隐藏。

## 当前开发进度

当前项目已经完成最小可玩闭环：

- 首页可以开始游戏。
- 观察阶段能播放动态事件。
- 事件会更新 `sceneState`、更新画面、写入 `eventLog`。
- 答题阶段的问题来自本局 `eventLog`。
- 答题后有反馈。
- 生命归零后进入结算。
- 可以重新开始。

当前仍是单机网页原型，没有接入任何外部服务。

## 后续计划

可以按小步继续扩展：

- 增加更多事件，例如招牌闪烁、门自动开合、店员移动、价格牌变化。
- 增加更多问题类型，例如组合记忆题、区域顺序题、数量变化题。
- 增加音效，例如监控电流声、倒计时声、答题反馈声。
- 增加每日挑战，使用固定随机种子生成同一套事件。
- 增加分享图，展示分数、完成轮数和最终评级。
- 优化库存模型，让货架商品、冰柜数量和物品显示更严格一致。
- 后续迁移到微信小游戏或抖音小游戏。

## 当前注意事项

- 当前不是最终美术效果，仍以 DOM、CSS、文字标签、色块为主。
- 新增事件时，要同时更新 `sceneState`、画面表现和 `eventLog`。
- 新增问题时，必须基于本局真实发生过的 `eventLog`。
- 不建议在当前阶段一次性大规模重构，优先保持原型稳定可玩。
