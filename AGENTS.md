# AGENTS.md

## 项目名称

《15秒记忆现场》

## 项目定位

这是一个手机竖屏网页小游戏原型。

核心玩法：玩家观察 15 秒动态场景，随后回答一个关于刚才画面细节的问题。

## 开发原则

1. 先做最小可运行版本，不追求商业级完整产品。
2. 使用 HTML + CSS + JavaScript，不接后端、不接登录、不接广告、不接平台 SDK。
3. 手机竖屏优先。
4. 不使用真实美术素材，先用 DOM、CSS、emoji、文字标签、色块实现。
5. 代码要清晰，方便后续迁移到微信小游戏/抖音小游戏。
6. 不要一次性做过多功能，每次只完成用户明确要求的部分。
7. 每次修改后必须说明：
   - 改了哪些文件
   - 如何运行
   - 如何验证

## 架构原则

1. 场景必须采用模块化分层元素系统，不要把场景做成一整张图片。
2. 场景分为以下层级：
   - 背景层
   - 人物层
   - 商品层
   - 信息层
   - 异常层
3. 所有顾客、商品、时钟、收银金额、监控屏、灯光效果都必须是独立可更新元素。
4. 每个事件都必须同时做三件事：
   - 更新 `sceneState`
   - 更新画面表现
   - 写入 `eventLog`
5. `QuestionGenerator` 只能基于 `eventLog` 生成问题，不能问本局没有发生过的内容。
6. 配置数据和游戏逻辑尽量分开，方便后续增加场景、事件、问题。

## 建议文件结构

```text
index.html
style.css
script.js
AGENTS.md
```

## 后续开发约束

后续每次开发都必须遵守本文件。

新增功能前应优先确认：

1. 是否符合“最小可运行版本”的阶段目标。
2. 是否只实现了用户本次明确要求的内容。
3. 是否保持手机竖屏优先。
4. 是否继续使用 DOM、CSS、emoji、文字标签、色块等原型级表现方式。
5. 是否保持场景元素的模块化和可更新性。
6. 是否让事件同时更新 `sceneState`、画面表现和 `eventLog`。
7. 是否让问题生成逻辑只依赖 `eventLog`。
