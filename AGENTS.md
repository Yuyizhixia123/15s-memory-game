# AGENTS.md

## 项目

- 名称：15秒记忆现场
- 类型：手机竖屏网页小游戏原型
- 核心玩法：观察便利店监控画面 15 秒，然后回答一个细节问题。

## 技术约束

- 只使用原生 HTML/CSS/JavaScript。
- 主要文件：`index.html`、`style.css`、`script.js`。
- 不接后端、登录、广告、排行、支付、平台 SDK，除非用户明确要求。
- 原型视觉优先使用 DOM、CSS、emoji、文字标签、色块和简单图形。

## 核心规则

- 不要修改核心玩法，除非用户明确要求。
- 每次只完成用户本次明确要求的内容。
- 手机竖屏优先。
- 不要把场景做成一整张静态图片。
- 配置数据和游戏逻辑尽量分开。

## 场景架构

- 场景必须保持模块化分层：
  - 背景层
  - 人物层
  - 商品层
  - 信息层
  - 异常层
- 顾客、商品、时钟、收银金额、监控文字、灯光、门、黑影都应保持独立可更新。

## 事件规则

- 每个事件必须同时完成：
  - 更新 `sceneState`
  - 更新画面表现
  - 写入 `eventLog`
- `QuestionGenerator` 只能基于本局 `eventLog` 生成问题。
- 不能询问本局没有真实发生过的内容。
- 不要破坏 `eventLog`、`sceneState`、`currentQuestion` 数据结构。

## 受保护系统

- 未经明确要求，不要重写：
  - `DifficultyManager`
  - `EventGenerator`
  - `QuestionGenerator`
- 未经明确要求，不要修改分数、生命、轮次、难度递增规则。

## UI 修改

- 视觉优化优先改 CSS 和展示层 DOM。
- 手机窄屏下控件必须清晰、可点、不遮挡主要画面。
- 不要添加复杂 UI，除非用户明确要求。

## 工作前后

- 修改前先简要说明计划和涉及文件。
- 修改后必须说明：
  - 改了哪些文件
  - 如何运行
  - 如何验证
